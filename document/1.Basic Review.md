# Basic Review

Mục tiêu của phần này là ôn lại các kiến thức cơ bản nhất trong Go trước khi đi vào các chủ đề lớn hơn. Sau khi học xong, bạn cần nắm được:

- Cách khai báo biến và hằng số.
- Các kiểu dữ liệu cơ bản trong Go.
- Cách Go suy luận kiểu, ép kiểu và `zero value`.
- Sự khác nhau giữa `type definition` và `type alias`.

***

## 1. Variables & Constants

### `var`

`var` dùng để khai báo biến theo cách minh tường. Bạn có thể khai báo kèm kiểu dữ liệu hoặc để Go tự suy luận giá trị ban đầu.

```go
package main

import "fmt"

func main() {
	var age int = 25
	var name = "Go"

	fmt.Println(age)
	fmt.Println(name)
}
```

Khi nên dùng `var`:

- Khi muốn khai báo biến ở `package scope`.
- Khi muốn dùng giá trị mặc định của biến.
- Khi cần viết rõ ràng kiểu dữ liệu.

### `:=`

`:=` là cách khai báo biến ngắn gọn, chỉ dùng được bên trong `function`. Go sẽ tự động suy luận kiểu từ giá trị bên phải.

```go
package main

import "fmt"

func main() {
	count := 10
	title := "basic review"

	fmt.Println(count)
	fmt.Println(title)
}
```

Lưu ý:

- `:=` không dùng được ngoài `function`.
- Ít nhất phải có 1 biến mới ở vế trái.

### `const`

`const` dùng để khai báo hằng số, nghĩa là giá trị không thay đổi sau khi được gán.

```go
package main

import "fmt"

const Pi = 3.14159
const AppName = "goneon"

func main() {
	fmt.Println(Pi)
	fmt.Println(AppName)
}
```

Nên dùng `const` cho:

- Các giá trị cố định như PI, `port` mặc định, `status code` tự định nghĩa.
- Các chuỗi cấu hình không thay đổi trong chương trình.

## 2. Data Types

### Numeric types

Go hỗ trợ nhiều kiểu số:

- Số nguyên: `int`, `int8`, `int16`, `int32`, `int64`
- Số nguyên không dấu: `uint`, `uint8`, `uint16`, `uint32`, `uint64`
- Số thực: `float32`, `float64`
- Số phức: `complex64`, `complex128`

```go
package main

import "fmt"

func main() {
	var a int = 10
	var b float64 = 3.5

	fmt.Println(a)
	fmt.Println(b)
}
```

Thông thường:

- Dùng `int` cho số nguyên thông thường.
- Dùng `float64` cho số thực để có độ chính xác tốt hơn `float32`.

### String

`string` là kiểu chuỗi trong Go. Chuỗi là `immutable`, nghĩa là không sửa trực tiếp từng ký tự trong chuỗi gốc.

```go
package main

import "fmt"

func main() {
	message := "Hello Go"
	fmt.Println(message)
}
```

### Boolean

Kiểu `bool` chỉ có 2 giá trị:

- `true`
- `false`

```go
package main

import "fmt"

func main() {
	isReady := true
	fmt.Println(isReady)
}
```

### Type inference

Go có khả năng tự suy luận kiểu dữ liệu từ giá trị được gán.

```go
package main

import "fmt"

func main() {
	x := 100      // int
	y := 12.5     // float64
	name := "Go"  // string

	fmt.Printf("%T\n", x)
	fmt.Printf("%T\n", y)
	fmt.Printf("%T\n", name)
}
```

Điều này giúp code ngắn gọn hơn, nhưng bạn vẫn cần hiểu rõ biến đang có kiểu gì.

### Type conversion

Go không tự động ép kiểu trong nhiều trường hợp. Bạn phải ép kiểu một cách minh tường.

```go
package main

import "fmt"

func main() {
	var a int = 10
	var b float64 = float64(a)

	fmt.Println(a)
	fmt.Println(b)
}
```

Ví dụ với chuỗi và số:

```go
package main

import (
	"fmt"
	"strconv"
)

func main() {
	number := 42
	text := strconv.Itoa(number)

	fmt.Println(text)
}
```

Lưu ý:

- `Conversion` khác với `parsing`.
- `float64(a)` là `conversion`.
- `strconv.Atoi("42")` là parse chuỗi thành số.
- `strconv.Itao(100) là parse số thành chuỗi`

### Zero value

Mọi biến trong Go nếu được khai báo mà chưa gán giá trị thì sẽ có `zero value`.

`Zero value` của một số kiểu phổ biến:

- `int` -> `0`
- `float64` -> `0`
- `string` -> `""`
- `bool` -> `false`

```go
package main

import "fmt"

func main() {
	var age int
	var name string
	var active bool

	fmt.Println(age)
	fmt.Println(name)
	fmt.Println(active)
}
```

Đây là một điểm rất quan trọng trong Go vì nó giúp biến luôn ở trạng thái hợp lệ ngay từ lúc được khai báo.

## 3. Custom Types

### Type definition

`Type definition` tạo ra một kiểu mới dựa trên kiểu có sẵn.

```go
package main

import "fmt"

type Celsius float64

func main() {
	var temp Celsius = 36.5
	fmt.Println(temp)
}
```

`Celsius` là một kiểu mới, không còn là `float64` thông thường theo nghĩa của hệ thống kiểu. Bạn có thể dùng nó để tạo ngữ nghĩa rõ ràng hơn cho dữ liệu.

### Type alias

`Type alias` tạo ra một tên khác cho một kiểu đã tồn tại, không tạo kiểu mới.

```go
package main

import "fmt"

type ID = int

func main() {
	var userID ID = 100
	fmt.Println(userID)
}
```

Trong trường hợp này, `ID` vẫn được xem là `int`.

### Underlying type

`Underlying type` là kiểu gốc bên dưới của một `custom type`.

```go
package main

import "fmt"

type Celsius float64

func main() {
	var temp Celsius = 30
	var raw float64 = float64(temp)

	fmt.Println(temp)
	fmt.Println(raw)
}
```

Ở đây:

- `Celsius` là `defined type`.
- `Underlying type` của nó là `float64`.

Phân biệt nhanh:

- `type Celsius float64`: tạo kiểu mới.
- `type Celsius = float64`: tạo alias.

***

## 4. Tóm tắt nhanh

Sau phần `Basic Review`, bạn cần nhớ:

- `var` để khai báo biến theo cách đầy đủ.
- `:=` để khai báo nhanh trong `function`.
- `const` dùng cho giá trị không thay đổi.
- Go có `zero value` cho mọi biến.
- Go cần ép kiểu minh tường, không tự động ép kiểu tùy tiện.
- `Type definition` tạo kiểu mới, còn `type alias` thì không.

## 5. Exercises

### 1. Temperature converter

Yêu cầu:

- Tạo `type` `Celsius` và `Fahrenheit`.
- Viết hàm đổi từ `Celsius` sang `Fahrenheit`.
- Viết hàm đổi ngược lại.

Gợi ý:

```go
type Celsius float64
type Fahrenheit float64
```

Công thức:

- `F = C*9/5 + 32`
- `C = (F-32)*5/9`

### 2. Currency converter

Yêu cầu:

- Tạo các biến thể hiện tiền VND, USD.
- Thử chuyển đổi giữa 2 đơn vị theo tỷ giá giả lập.
- In kết quả ra màn hình.

Ví dụ:

- `1 USD = 25000 VND`

Mục tiêu bài này là luyện:

- `Numeric types`
- `Type conversion`
- `Custom type`

### 3. Type alias practice

Yêu cầu:

- Tạo `type UserID = int`
- Tạo `type OrderID = int`
- Tạo thêm một `defined type` như `type Celsius float64`
- Thử gán giá trị và so sánh sự khác nhau trong cách sử dụng

Mục tiêu bài này là để thấy rõ:

- `Alias` chỉ đổi tên.
- `Defined type` tạo ra một kiểu riêng biệt.
